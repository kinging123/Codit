Introduction

My name is Reuven Karasik, I'm a programmer, and I made Codit. I'm 15 years old.

If you have ever tought any kids (8-16) programming, I'm sure you've faced at least one of the following:

  • The kids don't understand something and give up instead of trying to learn about it.
  • The kids feel like the subject is not interesting enough, as they can't see the result immidiatly.
  • The kids are having hard time memorizing all of the commands and syntaxes.
  • The kids write a lot of code that doesn't do anything, just because they need to.

    Yeah, I mean you, public static void main () !

Codit is here to solve all of those. It's a programming language built with JavaScript and runs localy, without Internet Connection, or any drivers needed*.

The Core Values behind Codit

  • Codit is very easy to use:

    All the commands are short and simple. (not System.out.println())

  • Codit is Super Intuitive: All the commands are just like we speak.

    You want to output something? use say something.
    If you know English, you can code :)

  • Codit is not qoute-sensitive

    Use strings, numbers and variables inline, without concatenating or using qoutes (" or ').

Uses of Codit in educational systems

Codit is not so powerfull as a software tool, but its power is unleashed when using it to teach kids and beginners how to program, very similar to Scratch.

The difference is, that Scratch uses blocks of "Lego", while Codit is an actual programming language. Scratch does not prepare you to the real "programming world": You don't have typos, you can hardly have bugs and it's all very fixed and you barely use your keyboard.

On the other hand, Codit is not that firm. All the errors you get are usually very self-explaining, sometimes there's even suggestions for the right syntax.
Unlike Scratch, you can make mistakes. You will. It's the best way of learning. The difference is, with Codit it's also easy to fix those mistakes.


More information about using Codit in your class: Installation, Customization, and the Educational Values

Basics

Command Structure

Every command is built out of mainly 2 parts:

  1. The command's name
  2. The command's properties

It looks like this: 

{command-name} {command-properties}

(example:) 

say hello world
in here, say is the command name, and hello world is the properties.

Remember that structure when Adding Your Own Commands

Variables and Variable names

In Codit, there's no qoutes or string concatenation, so all the literal values are written the same. To make sure the compiler can detect variable names instead of just string/integer, Variable Names MUST START WITH AN @ SIGN, like this: @vname.

Values and mathematic operations

Also, because there's no qoutes or string concatenation, you have to seperate values (strings, operators and variables) with spaces ( ). For example, let's say the variable @name contains the string "Reuven". To get the string "Hello, Reuven", you'll need to type Hello, @name. Pretty easy, isn't it?

A piece of code that "means" a value, such as Hello, @name is called a Value or a Phrase.

Math

Math Operators are operators that are used to make a mathematical calculation on a number (or a variable that contains a number).

The avaliable Math Operators:

  • +

    The Plus Operator: 6 + 3 will return 9

  • -

    The Minus Operator: 6 - 3 will return 3

  • *

    The Multiply Operator: 6 * 3 will return 18

  • /

    The Devision Operator: 6 / 3 will return 2

  • ()

    The Braces Operator: ( 6 + 2 ) * 3 will return 24

Math and math operators in Codit have to be seperated by spaces, just like any other values inside a phrase.

Conditions

When using Conditions, all JS conditions are valid, but it's better to use ==, !=, <, >, <= and >= like so: 

@var1 == @var2
(notice the spaces)

Built In Commands

The say Command

So.. if, somehow, you havn't noticed yet, Outputing is done with the say command.

Command Structure

say {text-to-output}

Code Examples

say hello
say hello world
say hello, world!
say hello, @name
say 3 * 5
say 4 - 8
say @score * 100 / 2

Note: by default, the output functionallity is based on the alert() method in JS. However, since we found that pretty annoying, in most workspaces there's a console for outputs and errors. But you can change it, obviously.

The set Command

The set Command is used for Setting Variables to values.

Command Structure

set {variable-name} to {value}

Code Examples

set @name to Reuven
set @test to Hello world!
set @score to 100
set @score2 to 25 * 3
set @avg to ( @score + @score2 ) / 2

Functions

Define a function using set

The set Command can be also used for Setting Variables to Functions. This means you can define a set of commands to be excecuted whenever you want how many times you want.

Note: currently, there's no option for passing arguments to the function. This is a feature that we will be adding soon. Stay tuned!

Command Structure

set {variable-name} to function:
{commands}
;

Code Examples

set @greeting to function:
say Welcome!
say Hello to my program
;

Calling a function using excecute

The excecute Command is used for calling a pre-defined Functions

Command Structure

excecute {variable-name}

Code Example

set @test to function:
say hello world!
;

excecute @test
excecute @test

Yeah, we know you're supposed to say Execute and not Excecute. We.. uh,.. Don't really know how to English. Sorry 'bout that. (Just kidding, it will be fixed in the next version)

Input from user using the get Command

The get Command is used for getting an input from the user into a Variable.

Command Structure

get {question} to {variable-name}

Code Example

get number? to @number
say The result is @number * 2

Note: by default, the input functionallity is based on the prompt() method in JS. You can change it, obviously.

Loops: the repeat Command

Creating loops with Codit is as intuitive as any other command. Just tell the computer to repeat!

Repeat Until (for loop)

Command Structure

repeat until {condition}:
{commands}
;

Code Example

set @i to 0
repeat until @i >= 10:
say hello world!
@i = @i + 1
;

Repeat As Long As (while loop)

Command Structure

repeat as long as {condition}:
{commands}
;

Code Example

set @i to 0
repeat as long as @i <= 10:
say hello world!
@i = @i + 1
;

Incrementing a variable

increment is used to increment a number (to add one to the original number). It becames extremely useful when using Loops This is just like the good old i++ from other languages.

Command Structure

increment @i

Code Example

(notice line 4) 

set @i to 0
repeat until @i >= 10:
say hello world!
increment @i
;

Flow Control: the if Command

Just like any other language, Codit has its own Flow Control syntax. But don't worry, you'll still get to use the good old if command.

Note: you can find more information about Conditions here.

Command Structure

if {condition}:
{commands}
;

Code Example

get Password to @password
if @password == 12345:
say Welcome!
;

if @password != 12345:
say Wrong Password!
;

Customize Codit

Note: as Codit is based on JS, Customizing Codit requires some basic JavaScript knowledge, as well as some experience in developing JS apps.

Customize Commands

Add Commands to Codit

One powerfull feature of Codit is the abillity to add new, custom, commands to the system. Those commands will function just like any other Built-In Commands, and you would be able to program the way they work using JavaScript

Every command you make, fits the Command Structure of Codit. You choose a Command Name and you pass a function that handels the Properties.

Now, let's code! This is how you make a function (The following code is JS code!): 

new Codit.cFunction(function(properties){
	// here goes the code that parses the properties
}, 'desctiprion', 'keyword');

The Codit.cFunction constructor accepts 3 parameters:

  • handler

    A function (function(){}) that accepts one parameter, which is the properties of the command, and parses them.

  • description

    A short piece of text that describes what the command does. e.g. Getting input from user

  • keyword

    The command's name: the first word of the line. e.g. say, get

This is an example of the say command: 

new Codit.cFunction(function(text){
	text = Codit.phrase(text);
	Codit.output(text);
}, "Printing", "say");
Learn more about Codit.phrase() and Codit.output() in Helper Functions.

Override Built-In Commands

As Codit is currently in Alpha stages, as for this version (0.3) there's no Filter for filtering Built In Commands. This is a feature that will probably be added in Codit 2.0, though we havn't started working on it yet. Have something to contribute? Contact us!

For now, if you want to override a Built In Command, you need to Regiser a New Command and COPY THE ENTIRE ORIGINAL CODE BEFORE MODIFING IT!. It is recommended to add a note that describes the act, like so: 

// We must override the set function of Codit. 
// This is the set Codit.cFunction for Codit 1.0. 
// If by using Codit.Visualize you lose some of the 
// original functionallity of the set Codit.cFunction,
// Please contact us and we will fix it ASAP.
(Of the Codit.Visualize files)

Customize the Workspace Environment

Your main HTML file can contain whatever you want, but it should link to the following files (In that order!):

  • jQuery

    The jQuery Library, you can download it from here.
    Note: this is optional, but it makes life easier and some Plugins, such as Codit.Visualize require it.

  • codit.compiler.js

    The main file of Codit, contains the compiler.

  • codit.functions.js

    The Built-In Commands of Codit.

  • codit.workspace.js

    Your Workspace File, which contains the linking between the HTML to the Codit Compiler.

  • codit.{plugin-name}.js

    (optional) The Plugin core file, if you use any plugins.

The codit.workspace.js File

One of the Required Files is the codit.workspace.js, which contains the linking between the HTML to the Codit Compiler.

The file must contain the Compiler Functionality, Output and input Functionality, and it could contain some Workspace Commands.

Note: it is recommended to load your code inside the workspace.codit.js file only when the page is ready. In jQuery, you can do it like this: 

$(function () {
// Your code
});

You can find an example of the file in the default workspace package.

Compiling the user's code

Obviously, the main functionality of your workspace is compiling the code. You can do it by a click of a button, or perhaps by a keyboard shortcut. Or both.

Codit provides you with a couple of compiling methods, both accept the code as a string. The first and most recommended one is Codit.execute(). That function uses try & catch to detect errors in the code and Output it nicely.

If you're testing your JS Code, maybe you'd want to use the Codit.compile() function. That function does not catch errors, but it means you can see the browser errors in the Developer Tools/Firebug box and make debbuging easier. Just don't forget to put it back to Codit.execute() before production, otherwise your user won't see his errors.

An example:
HTML: 

<textarea name="code" id="code" cols="100" rows="50"></textarea>
<br />
<input type="button" value="compile!" id="compile" />
JS (jQuery required): 
$("#compile").click(function () {
	Codit.execute($("#code").val());
});

$("#code").on("keydown", function (e) {
	if(e.keyCode == 13 && e.ctrlKey) { // Ctrl+Enter to excecute the code
		Codit.excecute($("#code").val());
	}
});

Output

Outputing to the user is an important step. The user must see errors, messages and feedback about his code as clearly as possible. A lot of Plugins will use the output functionality, so make sure you include it in your workspace.

Outputing is done using Codit.output(), which accepts the content, and also an optional parameter for displaying tips and notes. (Learn more about Codit.output() and other helper functions) This function is pre-defined to alert() the content. As we said earlier, this could be a bit annoying, so we recommend you display it nicely in the DOM.

To write the Codit.output() functionality, you need to create (override) that function. Here's an example:
HTML: 

<div id="output"></div>
JS: 
Codit.output = function (content, tip) {
	$("#output").append("<br /> > "+content);
	if(typeof tip !== "undefined") {
		$("#output").append("<small>("+tip+")</small>");
	}
}

Note: one of the specialities of alert() is the ability to stop the code execution, a thing you may want to keep.

Getting input from the user

Just like output, input is very important. The user should have feedback from his users, so it's important to include that functionality.

Inputting is done using Codit.input(), which accepts one parameter (the question) and returns the user's answer. (Learn more about Codit.input() and other helper functions).

The default inputting method is by the prompt() JS method. As for now, in version 0.3 there's no support for different types of input, because the speciallity of prompt() is that it returns an answer immidiatly. Maybe in the next versions there will be support for a different function.

Anyway, here's the code: 

Codit.input = function (content) {
	return prompt(content);
}

Workspace Commands

Sometimes you want your workspace to have it's own commands, in addition to the original ones. This is, of course, optional, and those functions should be in the workspace.codit.js file. Here's an example of 2 functions you may want to include: 

// Since we overridden the output functionality, 
// we want to have another way of 
// stopping the code excecution with an alert.
new Codit.cFunction(function(parameters){
	alert(parameters);
}, "Debugging Alert", "alert");

// Clearing function is very good for testing
new Codit.cFunction(function (parameters) {
	$("#output, #code").html("").val('');
}, "Clearing", "clear");

Helper Functions in JS

Helper Functions are JS functions that help you, as a developer, build plugins more simply.

Codit.output() - outputting

Codit.output() is used for displaying messages to the user. Customizing Output

Example: 

Codit.output('Hello world.');

Codit.input() - getting input

Codit.input() is used for getting messages from the user. Customizing Input

Example: 

name = Codit.input('What is your name?');

Codit.phrase() - evaluate

Codit.phrase() is used for getting the final value of a code bit.

Example: 

content = "@score + @score2";
content = Codit.phrase(content);

Codit.value() - getting value

Codit.value() is used for getting the final value of one literal - a variable, for instance.

Example: 

thePassword = Codit.value('@password');

Note: as you can never know what the user would enter, it is very recommended to use Codit.phrase() instead.

Codit.error() - throwing errors

Codit.error() is used for displaying an error to the user.

Example: 

Codit.error('Wrong syntax: wrong variable name', 'tip: remember you must start the variable name with an @ sign!');

Codit.compile() - compiling lines of code

Codit.compile() is used for compiling few lines of code.

Example: 

Codit.compile('say hello world' + "\n" +
'say goodbye');

Codit.query() - compiling bit of code

Codit.query() is used for compiling a single line of code.

Example: 

Codit.compile('say hello, @name');

Note: as you can never know how many lines the user would enter, it is very recommended to use Codit.compile() instead.

Plugins for Codit

If you wish, you can combine all above into a Plugin. What this means, is basically a single JS file, sometimes followed by a customized workspace. The JS file usually contains some new commands and/or overridden commands, as well as some functionallity. This is very up to you.

A nice example of a plugin is Codit.Visualize, which turns Codit into a fully-visualized programming language, just like scratch. You can check it out here.
